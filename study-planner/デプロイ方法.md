# 息子さんに渡す用：デプロイ方法（保護者向け）

息子さんが **Cursor や開発環境なしで**、ブラウザだけで使えるようにする手順です。Mac でも iPhone でも使えます。

---

## リポジトリを Private のままにしたい場合

GitHub で **リポジトリを非公開（Private）のまま** にしておく場合の選び方です。

| 方法 | ソースコード | アプリのURL | おすすめ度 |
|------|-------------|-------------|------------|
| **A. Vercel でデプロイ** | 非公開のまま（GitHub が Private でOK） | 発行されたURLだけ知っている人が開ける。検索には出ない | ◎ 手軽 |
| **B. ローカルのみ（公開しない）** | 非公開のまま。息子にはビルド済みフォルダだけ渡す | なし（息子の Mac のブラウザで localhost のみ） | ◎ 一切公開したくない場合 |
| **C. GitHub Pages** | **Public リポジトリでないと無料では使えない** | 誰でもURLを知っていれば開ける | × Private のままなら不可 |

- **「URL は息子だけに教える」でよい** → **A. Vercel** がおすすめです。リポジトリは Private のまま、Vercel に「Private リポジトリへのアクセス」を許可するだけです。
- **「ネット上に一切出したくない」** → **B. ローカルのみ** にしてください。息子の Mac でだけ動かします。

以下、A と B の手順を書きます。GitHub Pages（方法2）は **リポジトリを Public にした場合** の選択肢です。

---

## 方法A: Vercel でデプロイ（リポジトリは Private のまま）

リポジトリを **Private のまま** でも利用できます。公開されるのは「Vercel が発行するアプリの URL」だけです。その URL を他人に教えなければ、息子さんだけが使えます。

1. **Vercel に登録**  
   https://vercel.com で「Sign Up」→ **GitHub でログイン**。

2. **「Add New Project」** で、GitHub の `eisei-project`（Private のまま）を選ぶ。  
   - 初回だけ「Vercel がリポジトリにアクセスする」許可を求められたら **許可** する。

3. **設定**
   - **Root Directory**: `study-planner` に変更（「Edit」で指定）
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

4. **Deploy** を押す。

5. 表示された URL（例: `https://eisei-project-xxx.vercel.app`）を **息子さんだけに伝える**。  
   この URL は検索には出ず、知っている人だけが開けます。

- 今後、GitHub の `main` に push するたびに、同じ URL が自動で更新されます。

---

## 方法B: 一切公開しない（息子の Mac でだけ使う）

**ネット上にアプリの URL を出したくない** 場合は、息子さんの Mac で「ローカル」のまま動かします。

### 親（あなた）がやること

1. ご自身のパソコンで、リポジトリの **study-planner** フォルダを開く。
2. ターミナル（または Cursor のターミナル）で次を実行して **ビルド** する。

   ```bash
   cd study-planner
   npm install
   npm run build
   ```

3. **study-planner フォルダの中の `dist` フォルダ** をまるごと ZIP にする。
4. その ZIP を息子さんに渡す（USB、AirDrop、メール、クラウドなど）。

### 息子さん（Mac）がやること

1. **Node.js を入れる**（まだなら）  
   https://nodejs.org から「LTS」をダウンロードしてインストール。

2. 受け取った **ZIP を解凍** し、中身の **`dist` フォルダ** を好きな場所に置く（例: 書類フォルダ内に「学習プランナー」フォルダを作り、その中に `dist` を入れる）。

3. **ターミナル**（アプリケーション → ユーティリティ → ターミナル）を開き、次を実行する（`$` は入力しない）。  
   ※ `dist` を置いた場所に合わせて `cd` のパスを変えてください。

   ```bash
   cd ~/Documents/学習プランナー/dist
   npx serve
   ```

4. 表示された **「Local: http://localhost:3000」** のようなアドレスをブラウザで開く。

5. 使うときは、**毎回「ターミナルで `cd …` → `npx serve`」** が必要です。  
   終わるときはターミナルで **Ctrl + C** を押して止めます。

- データは **息子さんの Mac のブラウザ** に保存されます。アプリの URL はインターネット上には一切出ません。

## （参考）GitHub Pages（※リポジトリが Public の場合のみ）

**注意**: GitHub の無料プランでは、**Private リポジトリでは GitHub Pages を利用できません**。  
リポジトリを **Public** にしたうえで使う場合の手順です。

1. **リポジトリの設定**
   - GitHub で `ksasnses/eisei-project` を開く
   - **Settings** → **Pages**
   - **Build and deployment** の **Source** で **「GitHub Actions」** を選ぶ

2. このリポジトリにはすでに `.github/workflows/deploy-pages.yml` が入っています。  
   `main` に push すると自動でビルドされ、次の URL で公開されます。

   **URL**: `https://ksasnses.github.io/eisei-project/**

3. この **URL を息子さんに伝えて**、「使い方.md」と一緒に渡してください。

- 初回は push から 1〜2 分ほどかかることがあります。  
  Settings → Pages で「Your site is live at …」と出ていれば成功です。

## 息子さんに渡すもの

- **方法A（Vercel）** の場合: **アプリの URL** ＋ **「使い方.md」**
- **方法B（ローカルのみ）** の場合: **ZIP で渡した `dist` の使い方**（上記「息子さんがやること」）＋ **「使い方.md」**（ブラウザで localhost を開く部分）

Mac の Safari や Chrome で開けば使えます。  
Vercel で URL を渡した場合は、スマホでも同じ URL を開けます。「使い方.md」の「スマホでアプリのように使う」でホーム画面に追加するとアプリのように使えます。  
方法Bの場合はスマホからは使えず、その Mac だけです。
