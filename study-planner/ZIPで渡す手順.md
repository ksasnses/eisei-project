# ZIP で渡す手順（一切ネットに公開しない方法）

アプリを **ビルドして ZIP にし、息子さんに渡す** やり方です。  
息子さんはその Mac のブラウザでだけ使います。URL は一切公開されません。

---

## 親（あなた）がやること

### 1. ビルドする

**Cursor を開いた状態で**、ターミナル（または Cursor のターミナル）で次を実行します。

```bash
cd study-planner
npm install
npm run build
```

- 初回は `npm install` に少し時間がかかることがあります。
- 最後に「built in …」と出て、**study-planner フォルダの中に `dist` というフォルダ** ができていれば成功です。

### 2. dist フォルダを ZIP にする

- **エクスプローラー**（Windows）または **Finder**（Mac）で、**study-planner** フォルダを開く。
- その中にある **`dist`** フォルダを **右クリック** する。
- **「圧縮」** または **「送る」→「圧縮（zip形式）フォルダー」** を選ぶ。
- 同じ場所に **`dist.zip`**（または `dist.zip` のような名前）ができる。

### 3. 息子さんに渡す

- その **ZIP ファイル** を、次のいずれかで渡す。
  - **AirDrop**（Mac 同士の場合）
  - **メール**（ZIP を添付）
  - **USB メモリ** にコピーして渡す
  - **Google ドライブ / iCloud など** にアップロードして、息子さんだけに共有

---

## 息子さん（Mac）がやること

### 1. Node.js を入れる（まだなら）

1. ブラウザで **https://nodejs.org** を開く。
2. **「LTS」** と書いてある方の **「Download」** を押してダウンロード。
3. ダウンロードした **.pkg** を開いて、画面の指示どおりにインストール（「続ける」で進めてよい）。
4. 終わったら **ターミナル** を一度閉じて、もう一度開く。

### 2. ZIP を解凍してフォルダを置く

1. 受け取った **dist.zip** を **ダブルクリック** して解凍する。
2. 中身の **`dist`** フォルダができるので、これを **わかりやすい場所** に移す。  
   例：
   - **書類** フォルダの中に **「学習プランナー」** というフォルダを作り、その中に **`dist`** を入れる。  
   → パスは `~/Documents/学習プランナー/dist` のようになります。

### 3. アプリを起動する

1. **ターミナル** を開く。  
   （ Spotlight で「ターミナル」と入力して起動してもよい。）
2. 次のコマンドを **1行ずつ** 入力して Enter を押す。  
   ※ `dist` を置いた場所に合わせて、**2行目** のパスを変えてください。

   ```bash
   cd ~/Documents/学習プランナー/dist
   npx serve
   ```

   - 別の場所に置いた場合は、例えば：
     - デスクトップなら: `cd ~/Desktop/dist`
     - ダウンロードなら: `cd ~/Downloads/dist`

3. しばらくすると、次のような表示が出ます。

   ```
   Local:   http://localhost:3000
   ```

4. **ブラウザ**（Safari や Chrome）を開き、アドレス欄に **`http://localhost:3000`** と入力して Enter。

5. 学習プランナーの画面が開けば成功です。  
   初回は「設定ウィザード」が始まるので、「使い方.md」を見ながら進めてください。

### 4. 終わるとき

- ターミナルで **Control + C** を押すと、サーバーが止まります。
- 次に使うときは、もう一度 **「3. アプリを起動する」** の 2〜4 の手順（`cd` → `npx serve` → ブラウザで localhost:3000）を繰り返します。

---

## 注意

- **データ**（設定・タスクの完了状況など）は、**その Mac のブラウザ** に保存されます。  
  ZIP や `dist` フォルダを上書きすると、データは消えません（ブラウザに残っています）。
- **アプリの更新**（あなたがコードを直したとき）は、もう一度 **親がビルド → 新しい dist を ZIP にして渡す** 必要があります。  
  そのとき、息子さんは **新しい ZIP を解凍して、同じ場所の `dist` を置き換え** ればOKです。  
  ブラウザのデータは、同じブラウザで開き続けていればそのまま残ります。
- この方法では **スマホからは使えません**。使うのはその Mac だけです。

---

## うまくいかないとき

| 症状 | 対処 |
|------|------|
| `npm: command not found` | Node.js を入れたあと、ターミナルを**一度閉じて開き直す**。まだ出る場合は、Node.js のインストールをやり直す。 |
| `cd` で「No such file」 | `dist` を置いた場所のパスが違う。Finder で `dist` フォルダを開き、そのフォルダをターミナルにドラッグ＆ドロップすると、パスが入力される。 |
| ブラウザで「接続できません」 | ターミナルで `npx serve` を実行したままか確認。止まっていたらもう一度 `npx serve` を実行する。 |
| 親のビルドでエラー | `cd study-planner` になっているか確認。`npm install` を先に実行してから `npm run build` する。 |

不明な点があれば、このファイルを見ながら親に聞いてください。
